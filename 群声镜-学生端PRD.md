# 群声镜 · 学生端 PRD

**产品名称：** 群声镜（学生端）
**文档版本：** v0.2
**文档状态：** Demo 原型设计基准版
**适用平台：** 手机 App / 微信小程序
**设计范围说明：** 本文档面向内部展示 Demo，非核心功能（危机干预、登录体系、资源内容管理等）暂不深入设计。

---

## 一、产品概述

### 1.1 产品定位

学生端是群声镜的数据采集侧与用户服务侧，核心目标有两个：

1. **采集真实情绪数据**：通过低门槛、轻量化的交互，引导学生主动记录心理状态，降低社会称许性偏差。
2. **提供有价值的个人回报**：通过 AI 生成的个人情绪镜像和对话陪伴，让用户感受到记录的意义，形成使用粘性。

> **隐私承诺（产品核心信任基础）：** 学生所有数据仅用于匿名化群体画像统计，个体内容不对管理端开放，不可追溯至个人。

### 1.2 目标用户

在校大学生。核心使用场景：情绪低落或兴奋时需要表达出口、睡前情绪复盘、压力积累时的自我疏导。

### 1.3 视觉风格基调

- **风格：** iOS 式清爽简洁，具有高级感
- **配色：** 整体柔和，白底为主，辅以低饱和度品牌色点缀（参考：浅蓝灰、暖白、薄雾紫等方向）
- **控件：** 大圆角卡片、细线分割、充足留白，动效轻盈克制

### 1.4 整体页面架构

底部导航栏共三个一级模块（Tab）：

| 顺序 | Tab 名称 | 图标建议 | 核心定位 |
|------|----------|----------|----------|
| 1 | 记录 | ✏️ 羽毛笔 | 情绪输入入口 |
| 2 | 镜像 | 🪞 波纹圆 | 个人情绪报告 + AI伴侣 |
| 3 | 资源 | 📚 叶片 | 心理支持资源示意 |

## 二、功能模块详述

### 2.1 模块一：记录

#### 2.1.1 页面结构

进入「记录」Tab 后，页面中央展示**两张并列入口卡片**：

| 卡片 | 名称 | 副标题 |
|------|------|--------|
| 左卡 | 心理状态打卡 | "用几步描述此刻的你" |
| 右卡 | 心情树洞 | "想说的都可以写在这里" |

**流程关联：** 心理状态打卡完成后，系统自动引导用户进入心情树洞补充文字记录（可跳过）。

---

#### 2.1.2 功能一：心理状态打卡

**交互方式：** 全屏独立页面逐步推进，页面间以横向滑动或淡入淡出切换（参考 iOS Onboarding 体验）。顶部显示进度条（Step 1/4）。

**Step 1 — 时间维度**

| 要素 | 内容 |
|------|------|
| 主标题 | "你想记录的是？" |
| 选项 | 「此刻的感受」/ 「今天整体感受」|
| 交互 | 两个大卡片单选，点击后自动进入下一步 |

**Step 2 — 情绪量级**

| 要素 | 内容 |
|------|------|
| 主标题 | "此刻大致是什么感觉？" |
| 选项（5 级） | 非常愉快 / 有点愉快 / 不悲不喜 / 不太愉快 / 很不愉快 |
| 交互 | 情绪滑动条或五级卡片，辅以渐变色（暖→冷）增强感知 |

**Step 3 — 情绪关键词**

| 要素 | 内容 |
|------|------|
| 主标题 | "哪些词最能描述你现在的感受？" |
| 交互 | 标签云多选（最多 5 个），支持自定义输入 |
| 预置词示例 | 焦虑、平静、疲惫、满足、孤独、期待、烦躁、开心、委屈、充实…… |

**Step 4 — 影响因素**

| 要素 | 内容 |
|------|------|
| 主标题 | "对你影响最大的是？" |
| 预置分类 | 学业 / 人际关系 / 恋爱 / 家庭 / 兴趣爱好 / 身体状态 / 其他 |
| 交互 | 图标卡片，可单选或多选 |

**完成页**

- 展示本次打卡摘要卡片，如："你现在有点疲惫，主要受学业影响"
- 底部隐私说明："已匿名记录，仅参与群体分析，不关联个人身份"
- 主按钮：**「继续，写点什么」**（进入心情树洞）
- 次要操作：「完成，先这样」（跳过树洞，返回首页）

---

#### 2.1.3 功能二：心情树洞

心情树洞可从记录首页直接进入，也可在打卡完成后被引导进入。

| 要素 | 说明 |
|------|------|
| 页面风格 | 全屏沉浸式文本输入，背景简洁，减少干扰 |
| 输入方式 | 文字键盘输入 + 语音转文字（麦克风图标，实时转写） |
| 字数建议 | 500字以内，超出后轻提示但不截断 |
| 匿名提示 | 输入框下方固定文案："这里的内容完全匿名，仅参与群体分析" |
| 提交后反馈 | 显示 AI 即时共情回复（1-2句，温暖语气，非诊断性）|
| AI 回复示例 | "今天好像有些沉，能把这些写出来，本身就很勇敢。" |
| 历史记录 | 用户可查看自己过往的树洞记录，仅本人可见 |

---

### 2.2 模块二：镜像

#### 2.2.1 页面结构

「镜像」Tab 主体为**个人情绪报告**，报告页底部内嵌「和 AI 聊聊」按钮，点击跳转至独立 AI 心理伴侣对话页。

#### 2.2.2 个人情绪报告

页面顶部提供「本周」/「本月」Tab 切换，报告内容从上至下：

| 模块 | 内容 | 可视化形式 |
|------|------|------------|
| AI 文字总结 | LLM 生成约 100 字情绪叙事，温暖第一人称视角 | 卡片文本 |
| 情绪趋势 | 近 7 天 / 近 30 天情绪量级变化 | 折线图 |
| 情绪关键词 | 本周期内高频情绪词汇 | 词云或气泡图 |
| 影响因素 | 各压力源占比 | 横向条形图或环形图 |
| AI 伴侣入口 | 固定在报告页底部 | 卡片按钮，副标题："关于这周，想聊聊吗？" |

**空状态：** 记录不足时显示："再记录几天，你的镜像就会逐渐清晰～"，附打卡快捷入口。

#### 2.2.3 AI 心理伴侣（对话页）

| 要素 | 说明 |
|------|------|
| 对话风格 | 共情倾听为主，温暖非评判，适时给出认知视角 |
| 上下文感知 | 读取用户近期打卡数据进行个性化回应 |
| 顶部说明 | "AI 伴侣不能替代专业心理咨询" |
| 输入方式 | 文字输入（Demo 阶段） |

---

### 2.3 模块三：资源

> **Demo 处理方式：** 静态示意页面，展示内容结构即可，不实现动态推荐逻辑。

| 分区 | 示意内容 |
|------|----------|
| 校内心理热线 | 心理中心联系方式卡片（示例数据） |
| 冥想练习 | 2-3 条引导式冥想卡片（标题 + 时长） |
| 科普文章 | 3-5 篇文章卡片（标题 + 标签，如"焦虑管理""睡眠改善"） |

## 三、页面流程总览

```
App 启动（Demo 跳过登录，直接进入主界面）
  │
  ├── Tab 1：记录
  │     ├── 入口页（两张并列卡片）
  │     ├── [打卡] Step1 → Step2 → Step3 → Step4 → 完成页
  │     │                                         └──「继续写点什么」→ 树洞
  │     └── [树洞] 输入页 → 提交 → AI 即时回复
  │
  ├── Tab 2：镜像
  │     ├── 情绪报告页（周 / 月切换）
  │     └──「和 AI 聊聊」→ AI 心理伴侣对话页
  │
  └── Tab 3：资源
        └── 静态示意页（热线 / 冥想 / 文章）
```

## 四、数据与隐私设计

| 原则 | 实现方式 |
|------|----------|
| 匿名化采集 | 上传内容不携带可识别个人身份的信息 |
| 差分隐私 | 底层引入差分隐私算法，群体报表无法反向还原个体 |
| 知情同意 | 首次使用展示数据用途说明，用户主动确认后方可使用 |
| 个体数据隔离 | 管理端仅可查看聚合群体趋势，不可查看任何个人内容 |
